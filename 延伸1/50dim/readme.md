# 實驗過程
* 以前我們在用這個演算法（ESA）優化問題時，每次要建估算模型，都得自己用手手動去猜要用Cubic還是Gaussian核心，這就像每次畫地圖要手動選用圓珠筆還是鉛筆一樣麻煩。
* 這次實驗中，我們加入了一個自動核選擇的機制。它的做法是：每次演算法收集到新的數據、準備畫出地形估算圖時，程式會用數學公式快速算一下：『如果現在用 Cubic 畫，誤差大概是多少？如果用高斯畫，誤差又是多少？』。接著，程式會自動挑選誤差最小、畫得最準的那支筆來建模型。
我們把這個會自動選筆的『自適應版本』和以前只用 Cubic 的『舊版本』，放在50維度的7個不同高難度地形(基準函數）上進行 PK，每個地形都隨機用5種不同的起點（種子）去跑1000次測試。結果發現，程式在平坦的山谷（如 Rosenbrock）會自動多用Cubic（佔比 73.6%），而在極其狹窄崎嶇的深盆地（如RHC2）則會自動多用高斯（佔比58.8%）
；最後的PK結果顯示，自動選筆的自適應版本在所有地形的優化效果都比舊版更好，我們將這些優化數據和比例整理成了圖表。」
